# License review — Hunyuan3D-Paint

**Status:** approved · **Decision date:** 2026-05-20 ·
**Signatory:** Ken Allred (zigrivers@gmail.com)

## Subject

[Hunyuan3D-Paint](https://github.com/Tencent/Hunyuan3D-2) is Tencent's
texture-painting model. It takes a 3D mesh (typically with vertex
colours only) and generates PBR texture maps (albedo, roughness,
metallic, normal) for it. Distributed under the [Tencent Hunyuan
Community License](https://github.com/Tencent/Hunyuan3D-2/blob/main/LICENSE).

This document records the review decision that ungates
`scripts/texture.sh --mode paint`, which was previously a deliberate
stub returning `status=error error=needs_license_review`.

## Use cases this approval covers

The asset pipeline produces 3D assets used in:

- **Grithkin** — independent game project (commercial intent)
- **GripCraft** — independent game project (commercial intent)
- **Personal / research** — one-off prints, experiments

This approval covers texture generation for assets shipped in these
projects, including commercial distribution via Steam, itch.io, or
similar platforms.

## License clauses reviewed

The Tencent Hunyuan Community License includes the following clauses
that were material to this decision:

### 1. Monthly Active User (MAU) threshold

The license requires a separate commercial license once a product
using the model's outputs exceeds **100 million MAU**.

**Risk assessment:** Both target games are indie projects. Reaching
100M MAU would constitute mass-market success roughly at the level of
Fortnite or PUBG; this is not a realistic near-term concern. If
either project approaches this scale, a renegotiation with Tencent
would be required — a problem worth having.

### 2. Region exclusions

Some versions of Tencent's license family restrict distribution to
specific regions. The current Hunyuan3D-Paint license **does not
contain region exclusions** at the time of this review (versioned
2024-11). Future license revisions should be re-reviewed.

### 3. Output ownership

Outputs (the generated textures and any meshes textured with them)
**belong to the user**, not to Tencent. Commercial use of outputs is
permitted within the MAU + use-case constraints above.

### 4. Use-case restrictions

The license prohibits use for:
- Generating content that is illegal under applicable law
- Generating CSAM or other clearly prohibited content
- Military / weapons applications
- Surveillance applications targeting individuals

**Risk assessment:** None of these apply to a game asset pipeline.
Standard prohibitions; no operational impact.

### 5. Attribution

The license requires attribution to Tencent for redistributed model
weights. **It does not require attribution in derivative works**
(i.e., shipping a game that contains textures generated by the model
does not require crediting Tencent in the game).

**Operational implication:** if the pipeline's setup guide ever
republishes the model weights, those republished weights must include
the license file. Today the pipeline downloads weights from Tencent's
distribution, so this clause is satisfied automatically.

## Approval terms

- License bucket: `commercial_threshold` (same as SF3D, SPAR3D)
- Approval is for **the current Hunyuan3D-Paint license as of
  2026-05-20**. Future license revisions require re-review.
- No wrapper-side restrictions imposed. Manifest entries are tagged
  with the bucket; audit trail exists for any output that wants to
  be filtered later.

## Re-review triggers

Re-open this review if any of the following change:

- Tencent revises the Hunyuan Community License.
- Either target project crosses 10M MAU (10× safety margin).
- The pipeline begins distributing outputs as standalone marketplace
  assets (different commercial use pattern than "shipped inside a
  game").
- Distribution to new regions becomes relevant (re-check for region
  exclusions).
